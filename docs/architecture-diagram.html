Shop Service Functions
  • GET /products - Browse catalog
  • GET /products?search= - Search
  • GET /products/:id - Product detail
  • POST /orders - Place order (Saga Step 1)
  • GET /orders - Order history
  • GET /orders/:id - Order tracking
⚙ Supplier Service ECS Task (Fargate) 0.25 vCPU | 512MB | Port 8080
Supplier Service Functions
  • CRUD /admin/products - Manage inventory
  • GET /admin/orders - View all orders
  • POST /admin/orders/:id/confirm (Saga Step 2)
  • POST /admin/orders/:id/cancel (Compensate)
  • POST /admin/orders/:id/payment (Saga Step 3)
  • Payment failure --> Auto rollback stock

Saga Pattern - Order Processing Workflow

Step 1: Create Order
Service: Shop
Action: Validate stock, insert order, deduct stock
Transaction: BEGIN ... COMMIT
Failure: Rollback, no changes
➜
Step 2: Confirm Order
Service: Supplier
Action: Update status pending --> confirmed
Reject: Cancel order + restore stock (compensating)
➜
Step 3: Process Payment
Service: Supplier
Action: Insert payment, update status --> paid
Failure: Cancel order + restore stock (compensating)
➜
Complete
Order status: paid
Payment recorded
Stock permanently deducted
